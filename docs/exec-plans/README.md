# Execution Plans

Execution plans are short-lived working documents for the current milestone.

- Put the current operational plan in `docs/exec-plans/active/`
- Move finished plans to `docs/exec-plans/completed/`
- Keep `PLANS.md`, `docs/plan.md`, and `docs/prompt.md` aligned

This repository starts with one active plan for bootstrap follow-through and
FR-001/FR-002 ingestion work.
